# aider-stack

> [!WARNING]
> LLM-generated personal, unpolished project.

A set of utility scripts to drive Claude Code inside a Kitty terminal window by watching files for [Aider-style](https://aider.chat) comment-based AI markers.

It works with Claude running in the same Kitty window.

## Installation

```bash
make install
```

Installs `ai-watch` and `ai-watch-toggle` to `$HOME/bin/` by default.

## Usage

1. Launch Claude Code in a kitty terminal (note its `$KITTY_WINDOW_ID`).
2. Run the watcher in the same window, (different 'kitty window'):
   ```bash
   ai-watch --kitty-win-id <ID> [DIR]
   ```
   Or toggle via the launcher script:
   ```bash
   ai-watch-toggle
   ```

## Configuration

A sample [kitty.conf](kitty.conf) is provided which you can append to your own configuration to bind the toggle script to a hotkey (e.g. `F4`).

The project also includes [SKILL.md](SKILL.md) to orient Claude Code when handling instructions generated by `ai-watch`.

### Edit Safety Hook
The installer configures a global safety hook script (`aider-stack-confirm.py`) in `~/.claude/settings.json` under `PreToolUse`. During an ai-watch dispatch, if Claude attempts to edit a file that was not listed in the dispatch, the hook blocks the edit (exit 2) and returns an explanatory message; Claude relays it so you can approve the edit in chat or add the file to the dispatch.

**Gotchas:**
- The hook enforces strict matching using the exact `EDIT: <path>` marker generated by `ai-watch`. However, this restriction *only applies to prompts generated by ai-watch* (which start with `aider-stack:`). If you manually chat with Claude to edit a file, the hook intentionally ignores it and allows the edit freely.

## Markers

- `AI!` (DO): Claude makes the change; the file is marked for edit.
- `AI?` (INLINE): Claude adds a reply comment tagged `AI?>` below the line.
- `AI??` (ASK): Claude answers in chat; the file is not marked for edit.
- `AI` (CONTEXT): Sent to Claude as information; the file is not marked for edit.

`ai-watch` removes every marker token (only the token, not the line) from the file *before* dispatching, so Claude never sees or touches the markers and they cannot re-fire. `AI!`/`AI?` mark the file editable (`EDIT:`); `AI??`/`AI` do not (`FILE:`).
