# aider-stack

Orient Claude for handling instructions generated by ai-watch.

## When to Use
- Triggered by running `/aider-stack` or when asked to "use aider mode".

## Instructions
1. **Check for Plan**: Check for `PLAN.md` or `docs/PLAN.md` in the project's root directory, which may differ from `$PWD` (e.g. `root=$(git rev-parse --show-toplevel); test -f "$root/PLAN.md" || test -f "$root/docs/PLAN.md"`). If either file exists, read its contents for context and keep it in mind, but **do not carry out the plan now** — only act on the AI markers in the dispatched file.
2. **Restrict Edits**: Make edits **ONLY** in the files explicitly mentioned for edit.
3. **Propose Other Files**: If you believe it is useful or want to edit other files, do not modify them. Instead, *suggest* using those files.
4. **Brief Output**: Keep all responses and explanations extremely brief.
5. **Suggest Next Step**: When finished, suggest the next logical step.

## Dispatch Sections
A dispatch marks the file either `EDIT:` (you may edit it) or `FILE:` (reference only — do not edit it). It lists items under the headings below. 
- **DO**: Make the change described at the referenced lines.
- **INLINE**: Insert a new comment line directly below the referenced line, using the file's comment syntax, of the exact form `AI?> <your reply>`.
- **ASK**: Reply to the question in the chat window. Do not modify the file.
- **CONTEXT**: Treat the block as information for your task. Do not modify the file.
